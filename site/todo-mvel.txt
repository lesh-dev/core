XSM code style
==============
make_smth - готовит строку, которую надо выхлопнуть
draw_smth - сама выводит строку

Рефакторинг:
* Заменить "new" в XDB на значение по умолчанию (прогрепать константы)
* в readtext обработка длины работает неправильно
* Запилить баг про HTTP 500 при обращении к index.php/bla-bla/bla
* ...checkbox_attr vs .. from_post ---> выпилить в api

XSM:

+ У курса есть тип -- факультатив, вариатив, практикум (обсудить, как мы представляем отдельные задачи прака).
+ У курса появляется понятие цикла, на котором он читался!

+ Если курс читается на 2 циклах, он формально разбивается на 2, т.к. сложность такого курса удваивается,
  соответственно и зачёты по нему должны быть разные (можно сдать 1 часть и не сдать 2). Это разумные требования
  к процессу (так гораздо проще формализовать вес).
  Таким образом, цикл становится просто числом (или строкой-дропдауном)
+ У курса ещё можно рассматривать понятие сложности (уровень -- уже есть).

Юзабилити админки:

* чекбокс скрытая страница
* улучшить редактор атрибутов (неудаляемые поля)

* мигалки на сохранение полей
